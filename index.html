// principles I build by
// princípios que guiam minha construção
01
Agent plans. Domain executes. Policy never delegates. The deterministic gate lives in domain service code — regardless of what any upstream agent decides.
O agente planeja. O domínio executa. A política nunca delega. A porta determinística vive no código do domain service — independente do que qualquer agente upstream decida.
02
Single source of truth or nothing. Any data with two authoritative sources is data in the process of diverging.
Fonte única de verdade ou nada. Qualquer dado com duas fontes autoritativas é um dado em processo de divergência.
03
Extensibility is a data problem, not a versioning problem. New segments register new codes — the API surface stays stable.
Extensibilidade é um problema de dados, não de versionamento. Novos segmentos registram novos códigos — a superfície da API permanece estável.
04
Every document must work at two levels. Architecture blueprints must also serve as executive presentations — and vice versa.
Todo documento deve funcionar em dois níveis. Blueprints de arquitetura também devem funcionar como apresentações executivas — e vice-versa.
